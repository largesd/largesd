============================================================
FORMULA SUMMARY — LARGE-SCALE BLIND DEBATE SYSTEM (v1.2)
============================================================

This summarizes all quantitative formulas and their parameters.

------------------------------------------------------------
1. CENTRALITY (AMPLIFICATION-RESISTANT)
------------------------------------------------------------

Raw distinct AU references:
  refs_i = number of DISTINCT AUs referencing canon item i

Log-scaled centrality:
  centrality_i = log(1 + refs_i)

Per-pool cap (within topic/side/type/stratum):
  centrality_i_capped = min(centrality_i, P95_pool)

Parameters:
  - refs_i: distinct AU reference count
  - log: natural log (published choice)
  - P95_pool: 95th percentile centrality within selection pool

Purpose:
  Prevent exponential amplification via repetition.

------------------------------------------------------------
2. SELECTION SCORE
------------------------------------------------------------

Deterministic selection score:
  S_i = f(centrality_i_capped,
          distinct_support_i_internal,
          AU_quality_proxy_i)

Where f is:
  - monotone increasing in each argument
  - published functional form (e.g., weighted sum or product)

Example canonical simple form:
  S_i = w1 * centrality_i_capped
      + w2 * log(1 + distinct_support_i_internal)
      + w3 * AU_quality_proxy_i

Parameters:
  - w1, w2, w3: published nonnegative weights
  - distinct_support_i_internal: internal distinct contributor count
  - AU_quality_proxy_i ∈ [0,1]

------------------------------------------------------------
3. RARITY SLICE SELECTION
------------------------------------------------------------

Low-centrality set:
  L = bottom X% by centrality_i_capped within pool

Budget split:
  Majority slice = (1 - ρ) * K
  Rarity slice   = ρ * K

Parameters:
  - X (e.g., 60%)
  - ρ (default 0.20)
  - K ∈ {K_E, K_N, K_A}

Purpose:
  Guarantee minority/low-centrality representation.

------------------------------------------------------------
4. TOPIC RELEVANCE WEIGHT
------------------------------------------------------------

Selected mass per topic:
  SelMass_t = (# selected canonical premises + arguments in topic t)

Primary weight:
  Rel_t = sqrt(SelMass_t) / Σ_u sqrt(SelMass_u)

Alternative sensitivity weight:
  Rel'_t = log(1 + SelMass_t) / Σ_u log(1 + SelMass_u)

Parameters:
  - SelMass_t ≥ 0
  - sqrt: square root
  - log: natural log

Purpose:
  Dampen verbosity effects.

------------------------------------------------------------
5. EMPIRICAL FACTUALITY COMPONENT
------------------------------------------------------------

Deterministic ternary mapping:
  SUPPORTED    → p_i = 1.0
  REFUTED      → p_i = 0.0
  INSUFFICIENT → p_i = 0.5 (undetermined token, not a probability estimate)

For selected empirical premises in (t,s):

Mean probability:
  F_{t,s} = (1 / n_E) * Σ p_i

Supported-only variant:
  F_supported_only_{t,s} =
      (1 / n_SR) * Σ p_i over SUPPORTED + REFUTED items
  If n_SR = 0, F_supported_only is null.

Insufficiency rate:
  insuff_rate_{t,s} = n_INSUFFICIENT / n_E

Parameters:
  - p_i ∈ {0.0, 0.5, 1.0}
  - n_E: number of selected empirical premises
  - n_SR: count of SUPPORTED + REFUTED

Sensitivity (per topic-side):
  delta_D_true  = D with all INSUFFICIENT premises → p = 1.0
  delta_D_false = D with all INSUFFICIENT premises → p = 0.0
  max_abs_delta_D = max(|delta_D_true|, |delta_D_false|)

------------------------------------------------------------
6. NORMATIVE ACCEPTABILITY COMPONENT
------------------------------------------------------------

For selected normative premises under Frame F:

  N_{t,s,F} = (1 / n_N) * Σ q_i

Parameters:
  - q_i ∈ [0,1]
  - n_N: number of selected normative premises

------------------------------------------------------------
7. REASONING STRENGTH COMPONENT
------------------------------------------------------------

For selected canonical arguments:

  Reason_{t,s} = median_j( score_{arg,j} )

Then aggregated over arguments:
  Reason_{t,s} = median over arguments of judge medians

Parameters:
  - score_{arg,j} ∈ [0,1]
  - median used for robustness

------------------------------------------------------------
8. COVERAGE COMPONENT (BINARY)
------------------------------------------------------------

For opposing selected arguments:

Per target k:
  label_k ∈ {0,1}

Coverage:
  Coverage_{t,s} = (1 / n_targets) * Σ label_k

Parameters:
  - label_k = 1 if direct rebuttal trace exists
  - n_targets: number of opposing selected arguments

------------------------------------------------------------
9. COMPONENT COMBINATION (PER TOPIC-SIDE-FRAME)
------------------------------------------------------------

Let components present be:
  C1 = F_{t,s}
  C2 = N_{t,s,F}
  C3 = Reason_{t,s}
  C4 = Coverage_{t,s}

Geometric mean with epsilon floor:

  Q_{t,s,F} =
    ( Π max(Ck, ε) )^(1 / m)

Where:
  - m = number of components present
  - ε = small floor constant (e.g., 0.01)

Arithmetic sensitivity:
  Q_arith = (1/m) * Σ Ck

Note: The epsilon floor is load-bearing. If a topic-side has all REFUTED
empirical premises, F = 0.0. Without ε, the geometric mean Q would collapse
to 0 regardless of how strong the normative or reasoning components are.
Do not remove the epsilon floor when using ternary p.

------------------------------------------------------------
10. OVERALL SIDE SCORE (PER FRAME)
------------------------------------------------------------

  Overall_{s,F} = Σ_t ( Rel_t * Q_{t,s,F} )

Parameters:
  - Rel_t normalized to sum to 1 across topics

------------------------------------------------------------
11. DECISION DIFFERENCE (PER REPLICATE)
------------------------------------------------------------

Per replicate r:

  D_{r,F} = Overall_{FOR,F,r} - Overall_{AGAINST,F,r}

Distribution across replicates:
  {D_{r,F}}

Confidence:
  confidence_F =
    max( fraction(D_{r,F} > 0),
         fraction(D_{r,F} < 0) )

Verdict:
  FOR       if CI(D_F) entirely > 0
  AGAINST   if CI(D_F) entirely < 0
  NO VERDICT otherwise

Parameters:
  - CI method published (e.g., percentile interval)

------------------------------------------------------------
12. BUDGET ADEQUACY INDICATORS
------------------------------------------------------------

Mass coverage ratio per topic-side:

  coverage_ratio_{t,s} =
    (selected reference mass) / (total canonical reference mass)

Top-k coverage:

  topk_coverage =
    (# of top-k central items selected) / k

Parameters:
  - k fixed and published
  - reference mass = sum of distinct AU refs

------------------------------------------------------------
13. PARTICIPATION ENTROPY (DIAGNOSTIC)
------------------------------------------------------------

Let p_i = share of contributions in bucket i:

  Entropy = - Σ p_i * log(p_i)

Published in bucketed form (not exact raw values if privacy risk).
