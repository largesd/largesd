Goal:
Update the email processor to verify signed submission tokens before accepting
posts, classify parse results for safe ack behavior, and pass the verified
user_id into DebateEngineV2.submit_post.

Why this matters:
Frontend-only enforcement is insufficient because users can email the processor
directly. The processor must be the final authority on token validity, payload
integrity, and sender identity.

3. Add token verification after parse acceptance
   After extracting submission, add:

     if self.config.email_submission_require_auth:
         if not submission.auth_token:
             # This should have been caught by parse_for_processor, but guard anyway
             self._handle_rejection(
                 mail, msg_id, submitter_email, subject,
                 "missing_token", ack_safe=False
             )
             return

         config = EmailSubmissionAuthConfig(
             secret=self.config.email_submission_secret,
             ttl_minutes=self.config.email_submission_token_ttl_minutes,
         )

         try:
             verified_user = verify_email_submission_claims(
                 config,
                 submission.auth_token,
                 submission,
                 submitter_email,
                 user_lookup=self.debate_engine.db.get_user_by_id,
             )
         except ValueError as exc:
             reason = str(exc)
             # Map ValueError reasons to ack_safe decisions
             ack_safe_reasons = {
                 "expired", "wrong_token_type", "debate_mismatch",
                 "submission_mismatch", "side_mismatch", "topic_mismatch",
                 "payload_mismatch", "email_mismatch", "unknown_user",
                 "inactive_user", "unverified_email",
             }
             ack_safe = reason in ack_safe_reasons
             self._handle_rejection(
                 mail, msg_id, submitter_email, subject,
                 reason, ack_safe=ack_safe
             )
             return
         except Exception as exc:
             # Malformed JWT, invalid signature, etc.
             print(f"[EmailProcessor] Token verification error: {exc}")
             self._handle_rejection(
                 mail, msg_id, submitter_email, subject,
                 "invalid_signature", ack_safe=False
             )
             return

         user_id = verified_user["user_id"]
     else:
         user_id = None

Acceptance criteria:
- v3 email with valid token is processed and saved with user_id.
- v3 email with missing token is rejected without ack by default.
- v3 email with expired token is rejected with ack.
- v3 email whose token debate_id does not match body is rejected.
- v3 email whose token submission_id does not match body is rejected.
- v3 email whose side/topic was edited is rejected.
- v3 email whose facts/inference/counter_arguments were edited is rejected
  (payload_mismatch).
- v1/v2 email is rejected when require_auth=true and allow_legacy=false.
- Forwarded email (different From header) is rejected (email_mismatch).
- Cross-debate token reuse is rejected.
- Unparseable spam is silently dropped (no outbound reply).
- Metrics/logs capture every decision with version_hint, reason_code, sender.

Guardrails:
- Do not send outbound acks for unparseable mail, malformed JWTs, or invalid
  signatures.
- Do not import Flask in the processor. Use the standalone auth helper.
- Keep SENDER_WHITELIST support as a secondary defense; do not remove it.
- Ensure user_id is passed to submit_post only after successful verification.
