Goal:
Replace the local unsigned v1 email body generation in new_debate.js with a
call to the authenticated POST /api/debate/email-submission-draft endpoint.
The preview panel and pending-post list must use the v3 body and mailto from
the backend.

Why this matters:
This is the bridge between the frontend auth gate (Task 01) and the backend
draft endpoint (Task 02). After this task, the browser no longer constructs
unsigned email submissions.

Files to read before editing:
- frontend/static/js/new_debate.js
- frontend/static/js/data_bridge.js

Write scope:
- frontend/static/js/new_debate.js

Implementation steps:

1. Make submitPost async
   Change the function signature from:

     function submitPost() { ... }

   to:

     async function submitPost() { ... }

2. Replace the email-mode path in submitPost
   The current email-mode path (near line 362) does:

     const mailto = DataBridge.buildMailtoLink(
       currentDebateId, currentResolution, side.value, ...
     );

   Replace that entire block with an API call to the draft endpoint:

     if (currentDebateSource === 'api') {
       // existing direct API post path stays unchanged
       ...
     }

     // Email mode: fetch authenticated draft from backend
     let draft;
     try {
       draft = await BDA.api('/api/debate/email-submission-draft', {
         method: 'POST',
         body: JSON.stringify({
           debate_id: currentDebateId,
           side: side.value,
           topic_id: topic.value,
           facts,
           inference,
           counter_arguments: counter || ''
         }),
         preferLiveApi: true
       });
     } catch (error) {
       if (error.status === 401) {
         Auth.redirectToLogin('posting-auth-required');
         return;
       }
       if (error.status === 429) {
         BDA.showStatus('Rate limit exceeded. Please wait before generating another draft.', true);
         return;
       }
       const code = error.payload?.code || error.code;
       if (code === 'EMAIL_DEST_MISSING') {
         BDA.showStatus('Email posting is not configured on this server. Please contact an administrator.', true);
         return;
       }
       if (code === 'DEBATE_NOT_AVAILABLE_FOR_POSTING') {
         BDA.showStatus(
           'This debate is available from the published cache, but the live posting server '
           + 'cannot validate it. Please choose a debate from the live server or contact an administrator.',
           true
         );
         return;
       }
       // Network / other error
       if (!error.status && !error.payload) {
         showStaticDeploymentMessage();
         return;
       }
       BDA.showStatus(error.message || 'Failed to generate email draft.', true);
       return;
     }

     const mailto = draft.mailto;
     const body = draft.body;

     if (!mailto) {
       BDA.showStatus('Could not generate email draft.', true);
       return;
     }

     // Show email preview panel
     const emailPanel = document.getElementById('email-preview-panel');
     const mailtoLink = document.getElementById('mailto-link');
     const bodyPreview = document.getElementById('email-body-preview');

     mailtoLink.href = mailto;
     bodyPreview.textContent = body;
     emailPanel.style.display = 'grid';

     // Add to pending
     BDA.addPendingPost({
       id: draft.submission_id || ('email-' + Date.now()),
       side: side.value,
       topic: topic.options[topic.selectedIndex].text,
       facts: facts,
       inference: inference,
       modulation_outcome: 'pending'
     });
     updatePendingPostsList();

     BDA.showStatus('Email draft generated. Open your email client to send the submission.');

   DataBridge.buildEmailBody should no longer be called from submitPost().

4. Update copyEmailBody
   copyEmailBody() currently reads from the preview panel textContent. That
   behavior is still correct because the preview panel now contains the v3 body.
   No change needed unless the element ID changed (it did not).

5. Register the async action handler
   At the top of new_debate.js, the action is registered with:

     BDA.registerAction('submit-post', submitPost);

   No change needed; BDA.registerAction accepts async functions.
