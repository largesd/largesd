IMPLEMENTATION STEPS
────────────────────
  RECOMMENDED: SQLAlchemy core (not full ORM) — keeps existing SQL, adds pooling.

  Option A — SQLAlchemy core (recommended):

    1. Add sqlalchemy>=2.0.0 to requirements

    2. In backend/database_v3.py or new backend/db_engine.py:

         from sqlalchemy import create_engine, text
         from sqlalchemy.orm import sessionmaker

         engine = create_engine(
             DATABASE_URL,
             pool_size=10,
             max_overflow=20,
             pool_pre_ping=True,
             echo=False
         )
         SessionLocal = sessionmaker(bind=engine)

    3. Use SessionLocal() context manager in routes, or create a get_db() dependency

    4. Keep raw SQL via session.execute(text("SELECT ...")) for complex queries

  Option B — sqlite3 connection pool (lighter, but more code to maintain):

    1. Create backend/db_pool.py:
         import sqlite3
         from queue import Queue

         class ConnectionPool:
             def __init__(self, db_path, pool_size=10):
                 self.db_path = db_path
                 self.pool = Queue(maxsize=pool_size)
                 for _ in range(pool_size):
                     self.pool.put(sqlite3.connect(db_path, check_same_thread=False))

             def get(self):
                 return self.pool.get()

             def put(self, conn):
                 self.pool.put(conn)

             @contextmanager
             def connection(self):
                 conn = self.get()
                 try:
                     yield conn
                 finally:
                     self.put(conn)

    2. Replace all sqlite3.connect() calls with pool.connection()
    3. Bundle transactions where multiple statements must succeed/fail together

ACCEPTANCE CRITERIA
───────────────────
  [ ] No connection-per-query pattern remains
  [ ] Pool size configurable via env var
  [ ] Multi-statement operations use transactions (rollback on error)
