================================================================================
  TASK 1.4: Add CSRF protection for HTML forms
  Phase: 1 — Critical Security Fixes (Week 1)
  Priority: 🔴 Critical | Effort: 1 day
================================================================================

CURRENT STATE
─────────────
  • No CSRF tokens on form submissions
  • Risk: Cross-site request forgery on state-changing endpoints

TARGET STATE
────────────
  • Double-submit cookie pattern OR flask-wtf integration
  • All HTML forms include CSRF token
  • API routes (AJAX) exempt if using proper CORS + auth headers

IMPLEMENTATION STEPS
────────────────────
  Option A — Manual double-submit cookie (lighter weight, RECOMMENDED):

    1. On session/login: set csrf_token cookie (Secure, HttpOnly=False, SameSite=Lax)

    2. On form render: inject <input type="hidden" name="csrf_token" value="...">
       where value matches cookie

    3. On POST/PUT/DELETE: validate header X-CSRF-Token == cookie value

    4. Skip validation for API routes with Authorization header (Bearer token)

  Option B — flask-wtf (if adopting WTForms later):

    1. Add flask-wtf>=1.0.0 to requirements
    2. Configure WTF_CSRF_ENABLED=True, WTF_CSRF_TIME_LIMIT=3600
    3. Add {{ csrf_token() }} to all Jinja2 templates (if using Jinja2)
    4. For static HTML, generate tokens via API endpoint /api/csrf-token

ACCEPTANCE CRITERIA
───────────────────
  [ ] All state-changing HTML forms include and validate CSRF token
  [ ] Missing/invalid CSRF token returns 403
  [ ] API routes with Bearer auth remain functional without CSRF token
