Task 05: Tests and Verification
===============================

Primary files:
  tests for scripts/dev_workflow.py, if an appropriate local test location
  exists or can be added cleanly.

Verification targets:
  acceptance/run_ui_acceptance.py
  scripts/dev_workflow.py
  backend/utils/middleware.py
  backend/routes/auth_bp.py


Goal
----
Add focused regression coverage for the harness/workflow changes and run the
smallest useful verification commands before the full browser acceptance suite.


Required Existing Test Runs
---------------------------
Run targeted backend checks first:

  python -m pytest tests/integration/test_csrf.py tests/integration/api/test_auth.py -q

These must continue passing because the implementation must not weaken backend
CSRF or auth behavior.


Suggested New or Updated Tests
------------------------------
1. scripts/dev_workflow.py port guard

   Add a small test for the new bind-based helper:

     - Bind a socket to 127.0.0.1 and an available test port.
     - Call assert_port_available("127.0.0.1", port).
     - Assert it raises RuntimeError.
     - Assert the message includes the port and "--port".

   Keep the test lightweight. It should not start Flask.

2. Acceptance admin seeding

   Add harness-level coverage if practical:

     - Seeded acceptance admin auth includes is_admin=True.
     - authenticate_browser_user injects a localStorage user object with
       is_admin=true.

   If this is awkward to unit-test without Playwright, document why and rely on
   the full acceptance run. The full run exercises the behavior through AC-5
   admin.html and AC-9 admin proposal acceptance.

3. AC-7 diagnostics

   Add or update assertion paths so failed register/login responses include:

     - HTTP status
     - response body
     - #error-alert text

   If direct unit coverage is impractical, verify through code review and the
   success-path acceptance run.


Full Acceptance Verification
----------------------------
Run:

  python scripts/dev_workflow.py acceptance --port 5080 --timeout 60

If the new stale-server guard reports port 5080 is occupied, stop the old server
or run:

  python scripts/dev_workflow.py acceptance --port 5081 --timeout 60


Expected Outcome
----------------
After Tasks 01-04:

  - AC-1 through AC-6 no longer fail with HTTP 403 during setup auth.
  - AC-7 either passes or fails with a precise register/login API response.
  - AC-8 through AC-11 no longer fail with HTTP 403 during setup auth.
  - Admin-dependent criteria use a seeded first-user admin token and
    localStorage user.is_admin=true.
  - The old HTTP 429 cascade remains prevented by ENABLE_RATE_LIMITER=false.
  - Snapshot-generating criteria complete because JobWorker remains enabled.
  - CSRF protection remains enforced in production behavior.
  - The workflow fails fast if a stale server already owns the selected port.


Do Not Merge If
---------------
Do not consider the implementation complete if:

  - /api/auth/register or /api/auth/login were exempted from CSRF.
  - Direct API setup sends X-CSRF-Token without the matching csrf_token cookie.
  - The setup/admin user is not is_admin=True.
  - localStorage "user" is missing is_admin:true for setup/admin criteria.
  - DISABLE_JOB_WORKER=1 is used for acceptance.
  - The full acceptance workflow is skipped without a clear reason.
