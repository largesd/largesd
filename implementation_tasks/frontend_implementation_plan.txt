1.3 Workspace 1: Human Review Queue (workspace-queue)
------------------------------------------------------------------------------

Purpose: List all canonical empirical premises flagged for human review.

Layout:  Two-column grid (copy admin-shell--policy layout):
         - Left sidebar: filters + summary stats
         - Right main: scrollable table of flagged premises

Sidebar contents:
  - Filter by flag type (multi-select checkboxes):
      ENTITY_AMBIGUITY
      POLICY_GAP
      CONTRADICTORY_TIER1_EVIDENCE
      HIGH_IMPACT_INSUFFICIENT
      HIGH_IMPACT_LLM_DIRECTION
      CAUSAL_COMPLEXITY
      SCIENTIFIC_SCOPE_OVERCLAIM
      LLM_VALIDATION_FAILURE
      CONNECTOR_FAILURE
      TEMPORAL_SCOPE_AMBIGUITY
      SOURCE_CONFLICT
      SCOPE_MISMATCH
  - Filter by status: All / SUPPORTED / REFUTED / INSUFFICIENT
  - Filter by evidence tier: 1 / 2 / 3
  - Summary stats (pills):
      Total flagged: <count>
      Pending review: <count>
      Reviewed no change: <count>
      Reviewed correction: <count>

Main table columns:
  | Flag(s) | Premise ID | Topic | Side | Status | p | Best Tier | Reason | Review |

Row actions:
  - "Open detail" → opens the fact-check audit workspace with this premise pre-selected
  - "Add review" → inline modal or form to submit a HumanReviewRecord

Review form fields (when adding a review):
  - Reviewer role (text, default "admin")
  - Review outcome (select):
      REVIEWED_NO_CHANGE
      REVIEWED_CORRECTION
      REVIEWED_POLICY_GAP
      REVIEWED_SOURCE_DISPUTE
  - Review note (textarea, min 10 chars)
  - Submit → POST /api/human-review (new endpoint, see backend plan)

Table row display rules:
  - Status pill colors: SUPPORTED=green, REFUTED=red, INSUFFICIENT=yellow
  - p displayed as exactly 1.0, 0.0, or 0.5 (mono font)
  - Tier displayed as T1/T2/T3 pill
  - Flags as small tag pills (e.g., <span class="pill warn">SCOPE_MISMATCH</span>)
  - If small-count suppression applies to a flag aggregate, show "<k suppressed"

Empty state:
  "No premises are currently flagged for human review. The deterministic
   synthesis engine has not raised any flags requiring review."

------------------------------------------------------------------------------

Purpose: Deep-dive into a single FactCheckResult with full replay metadata.

Layout:  Two-column grid:
         - Left sidebar: premise selector (searchable dropdown + recent list)
         - Right main: audit panels

Right main panels (stacked vertically):

  A) Authoritative Result Header
     Row of pills:
       - Premise ID
       - Snapshot ID
       - Status + p
       - Best tier
       - Authoritative hash (truncated, with copy button)
       - Display summary hash (truncated, with copy button)
       - Merkle root (truncated, with copy button)

  B) Claim Expression Tree (collapsible, default open)
     Visual tree showing:
       Root node type (AND, OR, NOT, IF_THEN, COMPARISON, QUANTIFIER, ATOMIC)
       Each ATOMIC leaf:
         - subclaim_id
         - text (truncated to 120 chars)
         - status + p
         - best_evidence_tier
         - insufficiency_reason (if any)
         - human_review_flags (if any)
     Use nested indented divs (not a table) for tree visualization.

  C) Evidence Items Table
     Columns:
       | Evidence ID | Source | Tier | Direction | Confidence | Relevance | Retrieved |
     Row details:
       - source_type (OFFICIAL_STAT, GOV_DB, SCIENTIFIC_DB, etc.)
       - source_tier (1/2/3)
       - direction (SUPPORTS, REFUTES, UNCLEAR, NEUTRAL)
       - direction_confidence
       - direction_method (DETERMINISTIC_STRUCTURED, LLM_CLASSIFIER, MANUAL)
       - relevance_score
       - retrieval_timestamp
       - connector_version
       - For decisive items: show decisive_quote_span (blockquote style)
       - For structured sources: show claimed_value vs source_value comparison result

  D) Synthesis Logic (collapsible)
     Display the SynthesisLogic object as a structured read-only panel:
       - status_rule_applied
       - policy_rule_id
       - decisive_evidence (list of evidence_ids)
       - contradictory_evidence (list of evidence_ids)
       - insufficiency_trigger (if any)
       - authority_ranking_applied (true/false)
       - claim_expression_node_type

  E) Replay Manifest (collapsible, default closed)
     Codebox with formatted JSON of ReplayManifest:
       - manifest_id
       - parameter_pack (decomposition version, connector versions, etc.)
       - input_hashes
       - authoritative_output_hashes
       - merkle_root

  F) Human Review Records (collapsible)
     Table of existing HumanReviewRecord objects for this premise:
       | Review ID | Role | Outcome | Note | Timestamp | Hash |
     If none: "No human review records exist for this premise."

  G) Non-Authoritative Display Summary (visually separated)
     Grayed/bordered panel labeled "Non-authoritative display summary"
     Shows the display_summary prose. Include a warning note:
     "This prose is generated after the authoritative result and cannot change
      status or p."
