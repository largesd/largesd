LSD v1.2 Gap Closure Plan for debate_system
Date: 2026-04-28

Goal
- Close every remaining gap between the current v3 implementation and the LSD v1.2
  requirements document (LSD req.txt, 534 lines, 22 sections).
- Keep the system usable while we upgrade: ship in vertical slices, each with
  API + UI + acceptance tests.
- Every slice ends with: backend implementation, API contract tests,
  UI rendering checks, acceptance criteria update.

Implementation Scope (Current Run)
- This implementation run includes only: Phase 2 and Phases 4-10.
- Phases 0, 0.5, 1, and 3 are out of scope for this run and must be tracked
  as deferred in the final compliance report (Phase 10).

Execution Rules
- No silent behavior changes: every policy/model/schema change gets a changelog
  entry and a formula version bump.
- Every slice ends with:
  1) backend implementation
  2) API contract tests
  3) UI rendering checks
  4) acceptance criteria update
- Use feature flags for high-risk transitions:
  - FACT_CHECKER_MODE: {simulated, perfect_checker, online_allowlist}
  - SCORING_FORMULA_MODE: {legacy_linear, v1_2_sqrt, v1_2_log}
  - FRAME_MODE: {single, multi}
  - COVERAGE_MODE: {leverage_legacy, binary_v1_2}

Formula Compliance Overlay
- Treat quantitative formulas as hard acceptance targets, not advisory notes.
- For every formula implementation, publish:
  - exact functional form
  - parameter values
  - parameter version id
  - snapshot-level provenance (which formula version produced this output)
- Add deterministic fixtures that assert formula outputs for known toy inputs.
- Add "formula drift" tests to detect accidental changes in score math.

     - REFUTED: contradiction_score > threshold_contradiction (0.70)

   - New FACT_CHECKER_MODE value: "perfect_checker".
   - Under perfect checker:
     - If sufficient evidence: p in {1.0, 0.0} (discrete, no intermediates).
     - If evidence truly unavailable: status=INSUFFICIENT, p=0.5.

     Tier 3 (limited/uncertain).

4. Add insufficiency sensitivity outputs (§13.1)
   - For each topic-side, compute:
     - D_if_insufficient_true: recompute D assuming all INSUFFICIENT
       empirical premises have p=1.0.
     - D_if_insufficient_false: recompute D assuming all INSUFFICIENT
       empirical premises have p=0.0.
